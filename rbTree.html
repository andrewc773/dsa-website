Red-Black Trees

What are they?

Red-Black trees are a special kind of self-balancing tree. The nodes contain an extra bit of information- their color: red or black.
In order to remain balanced, the tree has several rules it follows to maintain itself, which it decides based on the color of the nodes.

The Rules

  1. The root must be black
  2. Children of red nodes are black
  3. All external nodes share the same black depth
  4. New nodes being inserted are always initially red
  5. No path can have two or more consecutive red nodes
  6. Null nodes are always considered black

Re-balancing

Red-Black trees self-balance by two mechanisms: color-flipping and rotations.

If the sibling of the new node's parent is:

  • black, rotate the tree. The colors should be reset so the parent is black and children are red.
  • red, color-flip. The new node's grandparent node is flipped, as well as both of its children (one of these children is the new node's parent).

Performance

Red-Black Trees on average perform at an O(log n) algorithmic complexity. Given that they are self-balancing, one can
assume they are always working at an optimized level.

Advantages

  • Self-rotating tree that allows for optimized peformance
  • Useful when there is a lot of data being added and removed

Disadvantages

  • B-trees are better for indexing and storing info on a disk
  • Implementation can be complicated; there are several differing operations for edge cases.
