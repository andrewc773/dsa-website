AVL Trees

What are they?

Invented in 1962, the AVL tree was the first self-balancing tree. The invariant to distinguish a balanced tree from an unbalanced tree was its height.

The rule to keep in mind for AVL trees is that the difference in height must be less than or equal to 1 (height<=1).


AVL trees self-correct when this rule is violated, using rotations. In code, this consists of redirecting pointers, which we will see later.

Comparison to Red-Black Trees

  • AVL trees tend to be more balanced than Red-Black trees, resulting in faster lookups
  • AVL trees perform slower insertions and removals than Red-Black trees due to more rotations to keep the tree balanced
  • AVL trees require more memory because the node class must store more information
  • AVL tree code for rotations tend to be more complex to implement


Overall, AVL trees are more useful in applications that are retrieval intensive, whereas Red-Black trees are better suited when data is being inserted and removed frequently. The latter is more often used in libraries in C++ and Java.
