Binary Search Trees

What are they?

Binary search trees are tree structures that order their nodes. A node whose key greater than the key of the node it is being compared to would belong to the
right of the compared node, and if the key is less than the compared node's key, it would go to the left.

The Advantages

Binary search trees have a faster average time complexity than their list counterparts. Assuming a tree is balanced, operations generally take O(log n) time compared to
lists or arrays that have a log(n) average time complexity.

  • Balanced search trees have an average search, deletion, and insertion time of O(log n)
  • The keys to our node are in sorted order, and can be accessed in multiple ways depending on the traversal order you implement.

The Disadvantages

  • The benefits of the binary search tree over structures such as linked lists and arrays are primarily assuming the tree is balanced. Unbalanced trees are similar in function
    and algorithmic cost to a linked list.
  • Binary search trees by themselves have no mechanism to self-correct any unbalancing that can occur from inserting or deleting nodes. This makes it unlikely for the tree
    to perform at its optimized time complexity after several insertion or deletions.

//This class defines basic binary search tree, and a method to add more nodes
//to the tree
public class BinaryTree<K extends Comparable<K>>{
                        
    private class Node {
        Node left, right; //left and right child for node
        K key; //data that node holds
                            
        Node(K key) {
            this.key = key;
        }
    }
                        
    Node root;
    int height;
    
    public BinaryTree(Node n) {
        root = n;
        root.left=null;
        root.right=null;
        height = 0;
    }
                        
    //returns true if node doesn't already exist and adding was successful
    public boolean addNode(K key) {
        if(root==null) {
            this.root = new Node(key); //if root is null, then begin list with new root
            return true;
        }
                            
            return addAux(root, key);
        }
                        
        private boolean addAux(Node n, K key ) {
            if(n.key.compareTo(key)==0) {
                return false;  //node already exists
                                
            } else if(n.key.compareTo(key)<0) { // if current node key is less than
                                                 //key, move right
                                
            if(n.right==null) {
                n.right = new Node(key); //new leaf node to right of current node
                return true;
            } else {
                return addAux(n.right, key);
            }
                                
            } else if(n.key.compareTo(key)>0) { //if current node key is greater than
                                                //key, move left
                if(n.left==null) {
                    n.left = new Node(key); //new node created to left of current node
                    return true;
                } else {
                    return addAux(n.left, key);
                }
            }
                            
            return false;
        }
    }
