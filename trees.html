Trees

What are they?

In computer science, a binary digit is the smallest unit of data for computing. Binary means that there are only two options: 0 or 1,
on or off, left or right, etc.

Binary trees are data structures that store data in a way that forms a tree-like formation. Each node is connected via an edge.
General binary trees do not need to have any specific order to them, but as we will see in later tree-structures, order is very important
to have a structure that can retrieve and store data efficiently.

Features of the tree

Each tree consists of nodes that represent a part of that tree. Nodes that have other nodes stemming from them are known as parents,
and their descendant nodes are called child nodes. Nodes who have no children are known as leaf nodes. The node that begins the tree,
where all other trees ultimately derive from, is the root node.

Any subspace of the tree is known as the subtree. The height of the tree is the number of edges from the root to the deepest node down the tree.

Complete Tree

Complete trees are trees whose levels are all filled at the maximum capacity with two children. The only parent nodes with less than two children should
be the last level, and the child nodes must be aligned to the left.

Full Tree

Full trees are trees with all nodes having zero or two children.

Performance

In optimal conditions, when a tree is balanced, search, insertion, and deletion time can average an O(log n) time. This is due to the nature of the tree's
structure. Every level you step down in the tree eliminates the half of the tree that you did not traverse. For example, starting from the root node, if you
traverse to the right child, you eliminate the left subtree from your choices. Cutting in half continuosly scales to a log2 n complexity.

Unbalanced trees can make performance suffer. A degenerate tree for example, is a tree where every parent node has only one child node. This results in a structure
similar to a linked list, and will perform to a worst-case of O(n) time.
